Building a SwiftUI Card Game with StoreKit In-App Purchases

This tutorial guides you through creating a SwiftUI-based card game app for iOS, focusing on implementing Apple’s StoreKit for in-app purchases. You’ll build an app where users can select from four decks (Classic, Personal, Funny, Dare) and view randomized card questions. The Classic Deck is free, while the others require a one-time purchase to unlock, but after they purchase it they can keep it forever. By the end, you’ll have a functional app with a clean UI and an easy in-app purchase system.

Overview

The demo app is a SwiftUI card game with a sleek gradient UI, where:

  • Users select a deck on the home screen, with icons corresponding to their deck type
  • The Classic Deck is free, while Personal, Funny, and Dare decks show a lock icon and require purchase to unlock.
  • Selecting a deck displays randomized card questions from JSON files in a card view.
  • StoreKit handles purchase prompts and unlocks decks seamlessly.

Overview video of final app: https://www.youtube.com/watch?v=19Zs8Gdee78

Getting Started

Set up your environment:

  • Xcode 16: Download from the Mac App Store or Apple Developer.
  • Swift 5.9: Included with Xcode 16.
  • iOS 18: Target deployment.
  • Simulator/Device: Use Xcode’s Simulator or an iOS device.
  • JSON Files: Add JSON files (classic_deck.json, personal_deck.json, etc.) to your project’s main bundle (with this basic format):

First, create Card.swift to define Card and Deck structs and load JSON data

This should look something like this:

import Foundation

struct Card: Identifiable, Codable {

    let id: Int

    let question: String

}

struct Deck: Codable {

    let deck: String

    let cards: [Card]

}

func loadDeck(deckName: String) -> [Card] {

    let fileName = deckName.lowercased().replacingOccurrences(of: " ", with: "_")

    guard let url = Bundle.main.url(forResource: fileName, withExtension: "json") else {

        print("Warning: \(fileName).json not found, returning placeholder card for \(deckName)")

        return [Card(id: 1, question: "Placeholder question for \(deckName)")]

    }

    print("Found JSON file at: \(url)")

    do {

        let data = try Data(contentsOf: url)

        let decoder = JSONDecoder()

        let deck = try decoder.decode(Deck.self, from: data)

        print("Loaded \(deck.cards.count) cards for \(deckName)")

        return deck.cards

    } catch {

        print("Error decoding JSON for \(deckName): \(error)")

        return [Card(id: 1, question: "Placeholder question for \(deckName)")]

    }

}

There are some important things to note about this code:

Import foundation: Foundation provides critical APIs: Bundle for accessing bundled JSON files, Data for reading file contents, JSONDecoder for parsing JSON, and String extensions (e.g., replacingOccurrences) for filename formatting. Without Foundation, you’d need low-level file handling or third-party libraries, complicating the code.

Functionality: Enables Bundle.main.url(forResource:withExtension:), Data(contentsOf:), and JSONDecoder, which are essential for loadDeck to locate, read, and parse JSON files.

Card Struct:

  • Properties:
  • Identifiable - preparing Card for SwiftUI’s collection views in potential future features, like a card deck. SwiftUI uses id to track items during UI updates, optimizing performance by redrawing only changed items.
  • Codable - allowing Card to be decoded from JSON. Swift auto-generates decoding logic since Int and String are Codable, mapping JSON { "id": 1, "question": "..." } to the struct’s properties.

Deck Struct:

  • Properties:
  • Codable
  • Not Identifiable - the decks will be selected (eventually - when we get to it) by currentDeckIndex in HomeView, so Identifiable is not necessary here like it is in the Card struct

loadDeck Function:

  • Functionality:
  • Takes a deckName (“Classic Deck”) and converts it to a filename that is (classic_deck.json) using lowercased().replacingOccurrences(of: " ", with: "_").
  • Uses Bundle.main.url(forResource:withExtension:) to locate the JSON file, returning a URL.
  • Reads the file into a Data object with Data(contentsOf: url).
  • Decodes the Data into a Deck using JSONDecoder().decode(Deck.self, from: data).
  • Returns the Deck’s cards array for use in CardView.

Now you are done with the setup of your app, and can finally implement StoreKit!

Implementing StoreKit requires a few steps, and follow them carefully:

Step 1: Enable In-App Purchases in Project Settings

  1. In the Project Navigator (left sidebar), click the project name at the top.
  2. Click the Signing & Capabilities tab at the top of the editor.
  3. Click the + Capability button in the top-left corner of the “Signing & Capabilities” section.
  4. In the search bar, type “In-App Purchase” and double-click In-App Purchase to add it to the target.
  5. Verify that “In-App Purchase” appears in the “Capabilities” section.

Step 2: Create StoreKit Configuration File - define non-consumable products for the Personal, Funny, and Dare decks, enabling local testing without App Store Connect.

  1. Create a new file, but instead of an Empty File, select File from Template…

  2. In the template chooser, scroll to the StoreKit section (or search “StoreKit”).

  3. Select StoreKit Configuration File and click Next.

  4. Name the file CardGameProducts.storekit, or something of that nature, and ensure it’s added to the “CardGame” target.

  5. Click Create. Verify that the file appears in the Project Navigator and opens in the editor.

Step 3: Add Products

  1. With CardGameProducts.storekit (Or whatever you named it) open, click the + button at the bottom left of your screen.
  2. Select Non-Consumable as the product type (since users buy a deck once and keep it forever.
  3. Configure the first product (Personal Deck):
  • Reference Name: Personal Deck (a human-readable name for your reference).
  • Product ID: (unique identifier, using your organization identifier). A good way to do this is to get your Bundle Identifier, which can be found by clicking on your project name in the top left, then it should show up in the main screen. You can use this as the layout for your name. Note: your Bundle Identifier most likely has a “-” in it, which is not allowed for the Product ID - so what you can do is name it as close to your BundleID as you can, while removing the “-”, and also, keeping everything lowercase makes it easy. For example, my Bundle ID is edu.gvsu.cis.StoreKit-Project, so I named my Product ID edu.gvsu.cis.storekitproject.personal_deck for the personal deck. Do not forget the “.personal_deck” (or whatever deck you are making) as the suffix!!
  • Price: Set to $0.99 (or your desired price).
  • Display Name: Personal Deck (shown to users during purchase).
  • Description: This is optional but you can do it if you want.
  1. Do the same thing for other products, such as Funny Deck, or Dare Deck, giving them each their own unique Reference name and Product ID.

Step 4: Configure the Scheme for StoreKit Testing

  1. Click the Scheme Menu in the top toolbar (near the Simulator name, e.g., “CardGame > iPhone 16 Pro”).
  2. Select Edit Scheme from the dropdown.
  3. In the scheme editor, select the Run configuration in the left sidebar.
  4. Click the Options tab at the top.
  5. Scroll to the StoreKit Configuration section.
  6. Click the which will currently say “None” and select CardGameProducts.storekit (This is the name of that file you created earlier) from the list.
  7. Click Close to save the scheme changes.

Step 4: Create StoreManager.swift - will handle StoreKit logic, including fetching products, processing purchases, and tracking entitlements.

This will just be an empty file again, no File from Template.

Your code will look like this:

import StoreKit

import SwiftUI

class StoreManager: ObservableObject {

    @Published var products: [Product] = []

    @Published var purchasedDecks: Set<String> = []

   

    private let productIds = [

        "edu.gvsu.cis.storekitproject.personal_deck",

        "edu.gvsu.cis.storekitproject.funny_deck",

        "edu.gvsu.cis.storekitproject.dare_deck"

    ]

   

    init() {

        Task {

            await fetchProducts()

            await updatePurchasedDecks()

        }

    }

   

    func fetchProducts() async {

        do {

            let products = try await Product.products(for: productIds)

            let updatedProducts = products

            await MainActor.run {

                self.products = updatedProducts

            }

        } catch {

        }

    }

   

    func purchase(_ product: Product) async throws -> Bool {

        let result = try await product.purchase()

        switch result {

        case .success(let verificationResult):

            switch verificationResult {

            case .verified(let transaction):

                await transaction.finish()

                await updatePurchasedDecks()

                return true

            case .unverified:

                return false

            }

        case .userCancelled, .pending:

            return false

        @unknown default:

            return false

        }

    }

   

    func updatePurchasedDecks() async {

        var purchased: Set<String> = []

        for await result in Transaction.currentEntitlements {

            if case .verified(let transaction) = result {

                purchased.insert(transaction.productID)

            }

        }

        let updatedPurchased = purchased

        await MainActor.run {

            self.purchasedDecks = updatedPurchased

        }

    }

   

    func isDeckPurchased(_ deck: String) -> Bool {

        switch deck {

        case "Personal Deck":

            return purchasedDecks.contains("edu.gvsu.cis.storekitproject.personal_deck")

        case "Funny Deck":

            return purchasedDecks.contains("edu.gvsu.cis.storekitproject.funny_deck")

        case "Dare Deck":

            return purchasedDecks.contains("edu.gvsu.cis.storekitproject.dare_deck")

        default:

            return true // Classic Deck is free

        }

    }

}

Of course, replace the Product IDs with your own product IDs.

Other important things to note about this code:

ObservableObject: StoreManager uses ObservableObject to notify the HomeView of changes to products and purchasedDecks, triggering UI updates, for example, showing “Play” vs “Unlock for …”.

Initializer:

  • Runs fetchProducts and updatePurchasedDecks in a Task to initialize StoreKit data when StoreManager is created.
  • Task enables asynchronous execution, and await ensures products and entitlements are ready before HomeView renders.
  • Async: StoreKit 2 uses async/await for network operations (fetching products, checking transactions), aligning with Swift’s concurrency model.

fetchProducts Function: Gets product details from StoreKit for the specified Product IDs.

  • Calls Product.products(for: productIds) to fetch products asynchronously.
  • Updates products on the main thread via MainActor.run to ensure thread safety, as @Published changes trigger UI updates.

purchase Function: Initiates a purchase for a given Product and verifies the transaction.

  • Calls product.purchase() to start the purchase, presenting a system purchase dialog.
  • Handles the result:
  • .success(.verified(transaction)): Marks the transaction as finished (transaction.finish()) and updates purchasedDecks, returning true.
  • .unverified: Returns false if verification fails (e.g., tampered transaction).
  • .userCancelled, .pending: Returns false for user cancellation or pending purchases (e.g., parental approval).
  • @unknown default: Returns false for future cases, ensuring forward compatibility.
  • Throws errors for StoreKit failures (e.g., network issues).

updatePurchasedDecks Function: Checks the user’s current entitlements to update the list of purchased decks.

  • Iterates Transaction.currentEntitlements to find verified transactions.
  • Collects product IDs in purchased and updates purchasedDecks on the main thread.
  • Ensures purchasedDecks reflects all purchased non-consumables, even from previous sessions.
  • Async: Entitlement checks involve StoreKit’s asynchronous APIs, requiring await.
  • MainActor: Updates @Published properties safely for SwiftUI.

isDeckPurchased Function: Determines if a deck is accessible (purchased or free).

  • Maps deck names to product IDs and checks purchasedDecks for presence.
  • Returns true for “Classic Deck” (free) and purchased decks, false for locked decks.
  • Switch: Clear mapping of deck names to IDs, avoiding dictionary lookups for a small, fixed set.

The changes made here should be pretty self explanatory because you are basically just modifying the view to take on the product IDs that we made. Here’s a short list of the things we changed to the HomeView and why:

Added StoreKit Import:

  • Change: Added import StoreKit alongside import SwiftUI.
  • Why: Enables access to StoreKit’s Product type for displaying product prices (e.g., $0.99) in the “Unlock” button.

Added @StateObject for StoreManager:

  • Change: Added @StateObject private var storeManager = StoreManager().
  • Why: Creates a StoreManager instance to manage products, purchases, and entitlements, enabling purchase logic and reactive UI updates when decks are unlocked.

Conditional Play/Unlock Button:

  • Change: Replaced the NavigationLink for “Play” with an if-else block:
  • If storeManager.isDeckPurchased(decks[currentDeckIndex]), shows NavigationLink to CardView with “Play”.
  • Else, shows a Button to initiate a purchase with storeManager.purchase, labeled “Unlock for (productPrice(for: decks[currentDeckIndex]))”.
  • Why: Controls deck access, allowing gameplay only for unlocked decks (Classic or purchased) and prompting purchases for locked decks, implementing monetization.

Updated deckEmoji Function:

  • Change: Modified to return 🔒 for locked decks if !storeManager.isDeckPurchased(deck), else returns the original emoji (🙂, 😂, 🫣, 😈).
  • Why: Visually indicates locked decks, improving UX by showing which decks require purchase.

Added productPrice Function:

  • Change: Added func productPrice(for deck: String) -> String to retrieve a product’s displayPrice (e.g., “$0.99”) or fallback to “$0.99”.
  • Why: Displays the localized price in the “Unlock” button, ensuring accurate pricing and App Store compliance.
